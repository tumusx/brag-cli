How it works

Commit → Document, in seconds.

Install the post-commit hook once and forget about it. Your brag document grows automatically with every delivery.

1

Name your branch

Use a Jira ticket in the branch name — feature/ABC-123-description. That's all the setup needed.

2

Commit normally

The post-commit hook fires on every git commit. No extra commands, no interruptions to your flow.

3

Jira data is fetched

brag-cli reads the ticket from the Jira API — summary, status, type, priority, story points, description.

4

Document is saved

One file per ticket, prepended with each new commit. Saved to Obsidian or Google Docs — your choice.

5

Organized by period

Documents land in a 2026-S1/ semester folder, a 2026/ year folder, or flat — configured once, applied forever.

6

Pushed to GitHub

Optionally publish Obsidian docs to a GitHub repo for a versioned, shareable history of your deliveries.

Integrations

Your tools. One workflow.

brag-cli connects the tools you already use — pick your storage backend and go.

⚡

Git

Reads commit hash, message, author, diff stats, changed files, and repository name. Installs a post-commit hook to automate everything.

post-commit hook
🔷

Jira

Fetches ticket metadata via REST API v3 — summary, status, type, priority, story points, labels, and description.

REST API v3
🟣

Obsidian

Creates and updates structured markdown files with YAML frontmatter in your vault. Organized in semester/ or year/ subfolders.

YAML frontmatter
📄

Google Docs

Saves documents directly to Google Drive via OAuth2. Creates folders by period, one doc per ticket — accessible from anywhere.

Drive API · Docs API
🐙

GitHub

Publishes Obsidian documents to a GitHub repo via the Contents API — a versioned, shareable history of your deliveries.

GitHub API v3

Install

Up and running in 2 minutes.

Node.js 18+ required. No extra dependencies.

Install & configure

# Install globally npm install -g brag-cli # Obsidian backend (default) brag config --obsidian ~/mind/brag # OR: Google Docs backend brag config \ --google-client-id YOUR_ID \ --google-client-secret YOUR_SECRET brag auth google brag config --storage googledocs # Choose folder structure brag config --folder-structure semester # Configure Jira brag config \ --jira-url https://co.atlassian.net \ --jira-email you@company.com \ --jira-token YOUR_TOKEN

Enable the hook

# Install post-commit hook cd your-project brag hook install # That's it — commit as usual git commit -m "feat: ..." # Or log manually brag log brag log --push # Check saved docs (grouped by folder) brag status
